# Hardware Reference — Yamaha DMP9 / DMP9-16

## CPU: Toshiba TMP68301AF-16

The TMP68301 is a Toshiba ASSP integrating a Motorola 68000 CPU core with on-chip peripherals.

- CPU core: Motorola 68000, 16 MHz — IC24 on DIG board (XN047A00)
- Integrated UART SC0 (used for MIDI at 31250 baud, 8N1) and SC1
- Integrated parallel I/O (panel scanning, LED drive)
- Integrated timer channels (3× 16-bit)
- Integrated interrupt controller
- External bus interface for ROM, DRAM, DSP, LCD

Internal peripheral base: `0xFFFC00` (SFR block).
Full register map: see `src/hw/tmp68301_regs.h` and Toshiba TMP68301 datasheet.

**Important:** The TMP68301's on-chip memory management registers (AMARO, AAMRO, AACRO)
are **not used** for chip-select decoding on this hardware. All address decoding is performed
by two external PLDs (PLD-TM1, PLD-TM2).

## Memory map

| Range | Size | Device | Description |
|-------|------|--------|-------------|
| `0x000000` | 512 KB | TMS27C240 (IC34) | ROM — `/ROMCE` from PLD-TM1 |
| `0x400000` | 64 KB | MSM511664-80 (IC35/IC36) | DRAM — `/RAMOEL`, `/RAMOEU` from PLD-TM1 |
| `0x460000` | 4 KB | YM3804+YM6007+YM3807+YM6104 | DSP EF1 registers — `/MEGWF`,`/WMEGRD` from PLD-TM2 |
| `0x470000` | 4 KB | YM3804+YM3807 | DSP EF2 registers — `/MEGWF`,`/WMEGRD` from PLD-TM2 |
| `0x490000` | 2 B | HD44780-compatible | LCD — `/LCD` from PLD-TM2 |
| `0xFFFC00` | 1 KB | TMP68301 internal | SFR: UART, GPIO, timers, interrupt controller |

### DRAM dual-use

The 64 KB DRAM at `0x400000` is shared between two consumers:

- **CPU stack:** SSP initialised to `0x0040FED0`; grows downward toward `0x400000`
- **DSP delay buffer:** grows upward from `0x400000`

Stack top `0x0040FED0` leaves approximately 48 KB of DRAM for DSP delay lines.

## ROM: TMS27C240-10 (XN349A00) — IC34

- Texas Instruments TMS27C240
- 4 Mbit (512 KB) OTP EPROM
- 10 ns access time variant
- 32-pin DIP package, 5 V supply

## Address decoding: PLD-TM1 and PLD-TM2

All chip selects are generated by two PLDs on the DIG board, confirmed from service
manual pages 35–36.

### PLD-TM1 (IC33 — XA353A00)

Inputs: A16–A19, AS, UDS, LDS, CS0, CS1, R/W from TMP68301 bus

| Output | Target | Address |
|--------|--------|---------|
| `/ROMCE` | IC34 EPROM | `0x000000` |
| `/RAMOEL` | IC35 DRAM (lower byte) | `0x400000` |
| `/RAMOEU` | IC36 DRAM (upper byte) | `0x400000` |
| `/LC0`, `/LC1` | LED drivers | (LED section) |
| `/REC` | Rotary encoder | (encoder section) |

### PLD-TM2 (IC32 — XA354A00)

| Output | Target | Address |
|--------|--------|---------|
| `/LCD` | HD44780 LCD | `0x490000` |
| `/MEGWF`, `/WMEGRD` | DSP MEG write/read | `0x460000` (EF1), `0x470000` (EF2) |
| `/KOC` | Key matrix (front panel) | — |
| `/AUXWR`, `/ROMAUX` | AUX section | — |

## LCD: HD44780-compatible

- Character LCD module, **4 lines × 16 characters** (confirmed from manual LCD screenshots)
- Optimised for above-eye-level reading (rack-mounted unit viewed from above)
- Instruction register: `0x490000` (write-only in firmware)
- Data register: `0x490001` (read/write)
- `/LCD` chip select generated by PLD-TM2

Display content is 16 characters wide (confirmed from LCD screenshots in manual).
All strings written via `lcd_write_cmd` (`0x0000238C`) and `lcd_write_data` (`0x000023DC`).

### HD44780 commands used by firmware

| Command | Byte | Effect |
|---------|------|--------|
| Clear display | `0x01` | Clear + cursor home |
| Cursor home | `0x02` | Cursor to (0,0) |
| Set entry mode | `0x06` | Increment cursor, no shift |
| Display ON | `0x0C` | Display on, cursor off |
| Function set | `0x38` | 8-bit, 2-line, 5×8 font |
| Set DDRAM address | `0x80+n` | Move cursor to address n |
| Write data | via DATA register | Write character at cursor position |

## MIDI interface

- Standard MIDI DIN-5 IN / OUT / THRU
- UART: TMP68301 SC0 — `SC0BUF` at `0xFFFC0F` region
- Baud rate: 31250 baud, 8N1
- RX: SC0 receive interrupt → MIDI message parser (`midi_process_rx` at `0x000128EC`)
- TX: `midi_tx_byte` at `0x000122E0`
- Messages: Program Change, Control Change, SysEx (Yamaha ID `0x43`)
- 671 controllable parameters, 16 banks × 96 CCs

## Effects engine

Two independent multi-effects processors, controlled via address region starting at
`0x460000` (EF1) and `0x470000` (EF2).

### EF1 block (0x460000) — `dsp_ef1_write` at `0x0000274E`

| IC | Part | Function |
|----|------|----------|
| IC19, 29, 46, 70, 71 | YM6007 (XF164A00) | DSP2 — main effects (reverb, delay, pitch) |
| IC26–28, 37–39 | YM6104 (XE788A00) | DEQ2 — programmable digital EQ |
| — | YM3804 | Core reverb/delay engine |
| — | YM3807 | LFO modulation source |

Supported effect types (EF1 only): HQ-Rev 1 Hall, HQ-Rev 2 Room, HQ-Rev 3 Stage,
HQ-Rev 4 Plate, Flange, Chorus, Phasing, Tremolo, Symphonic, Early Ref.1, Early Ref.2,
Gate Reverb, Reverse Gate, Delay L–C–R, Stereo Echo, Pitch Change.

### EF2 block (0x470000) — `dsp_ef2_write` at `0x00002904`

| IC | Part | Function |
|----|------|----------|
| — | YM3804 | Reverb/delay |
| — | YM3807 | LFO modulation |

Supported effect types (EF2): Rev1 Hall, Rev2 Room, Rev3 Stage, Rev4 Plate, Flange,
Chorus, Phasing, Tremolo, Symphonic, Early Ref.1, Early Ref.2, Gate Reverb, Reverse Gate,
Delay L–C–R, Stereo Echo, Pitch Change.

EF1 and EF2 can operate in **Individual** mode (independent sends) or **Serial** mode
(EF1 feeds EF2). Controlled via `"-- Eff.Assign --"` LCD screen.

### DSP coefficient tables

Coefficient tables at ROM `0x059B00–0x06283F` are **byte-identical** across all three ROM
versions. They represent pre-computed DSP parameters (reverb coefficients, filter tables,
delay tap weights). Extracted once and shared as `src/shared/dsp_tables.S`.

## Front panel (key matrix + encoders)

- Rotary encoders for level and parameter adjustment
- Pushbuttons for scene recall, function selection, EF1/EF2, MIDI, etc.
- LED indicators for channel status, metering (7-segment bargraphs on SEND 3/4)
- MEMORY indicator: 2-digit 7-segment display for scene number; dot = MIDI RX indicator
- Key matrix scanned via TMP68301 GPIO + `/KOC` from PLD-TM2
- Encoders via `/REC` from PLD-TM1
