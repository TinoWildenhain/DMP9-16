# Version diff: v1.10 → v1.11

**XN349F0** (1994-01-20) → **XN349G0** (1994-03-10)

## Summary

| Metric | Value |
|--------|-------|
| Changed bytes | 171,097 (32.6% of 512 KB) |
| Changed regions | 96 (merged, gap ≤ 128 bytes) |
| Nature of change | Targeted bugfix/patch release — many small scattered changes |

This is a **minor revision**. The large number of small scattered regions (96 regions,
many 1–2 bytes) is characteristic of:
1. Scattered `JSR`/`BRA` target address fixups throughout code (1–2 byte offset changes)
2. Small function replacements in the `main_code` region
3. Version string update (0x05059A–0x05062E)

The one large region (0x012A8F–0x03F50D = 182 KB) in `main_code` indicates a
block of code was relinked, probably due to a function inserted or removed
earlier in the binary shifting all subsequent addresses.

- Multiple ~100–900 byte blocks changed (bug fixes or feature additions)
- Many 1–2 byte changes (branch/call target fixups)

## Key observations
- Version string is the most clearly identifiable change
- No data/coefficient table changes (0x059B00+ region identical) — presets unchanged
- The scattered 1-byte changes are almost certainly JSR/BSR offset fixups from
  inserting/removing code earlier in the binary
- Diagnostic version string at 0x053E1C updated ("DIAGNOSIS v1.2" → check v1.10)
